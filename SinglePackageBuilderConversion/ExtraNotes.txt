Detecting “real new builds” externally

Workflow runs endpoint:
GET https://api.github.com/repos/r-universe/tempbioc/actions/workflows/build.yml/runs

Each run has:
  id (unique per run)
  head_sha (commit that triggered the run)
  event (push, workflow_dispatch, etc.)
  status (completed, queued, in_progress)
  conclusion (success, failure, cancelled)

Architecture for external monitoring

Because you cannot run code inside R-universe:

    Monitoring script runs in your repo (or a CI you control):

        Polls the R-universe build workflow API every few minutes, or is triggered by a webhook if available.

        Checks head_sha vs last recorded SHA.

        Downloads DESCRIPTION at that SHA.

        Compares version to previously recorded version.

        Updates the stored version/commit info.

        Posts notifications if a new build/version is detected.

    Storage for previous state:

        Could be a small JSON file committed to your repo, e.g. r-universe-builds.json:

        {
          "dplyr": {
            "last_sha": "abc123",
            "last_version": "1.2.0"
          },
          "ggplot2": {
            "last_sha": "def456",
            "last_version": "3.4.1"
          }
        }
